# XinovaSU 独立落地页设计规格

## 目标

在当前空目录创建一套独立的静态落地页，用清晰、克制的白色技术编辑风格介绍 XinovaSU 的内核级 Android Root 能力。页面必须提供桌面端和移动端两套经过专门编排的响应式体验，而不是把桌面布局机械缩小。

## 已确认的视觉方向

- 基调：纯白背景 `#FFFFFF`、近黑文字、冷灰分隔线；仅以冰蓝 `#2563EB` 表达激活、连接与状态。
- 气质：高级技术杂志与工程图纸，而非赛博朋克、黑客霓虹或常见的圆角卡片堆叠。
- 图形语言：细网格、极细技术连线、单色线性图标和抽象的内核/LKM 架构示意图。
- 排版：大字号中文标题、严谨栅格、宽阔留白；导航及注释使用紧凑的无衬线字体和少量等宽字体。
- 视觉基准：已生成的首屏、能力、功能、技术亮点四张白色概念图；实现时以其版式、色彩与密度为准，不把图片中的非精确文案作为网站文案来源。

## 内容与信息架构

1. 顶部导航：XS/XinovaSU 标记，锚点“能力”“特性”“技术亮点”，以及 GitHub 和“下载最新版”。
2. Hero：XinovaSU、主张“内核级 Android Root —— 更深、更隐蔽、更可控”、简介和下载/GitHub/使用文档三个 CTA；配抽象的白色内核模块装置图。
3. 简介与核心能力：产品定义、Android 12/5.10 至 Android 16/6.12 的兼容范围，以及四项能力：内核级 SU 与鉴权、模块系统 + WebUI、挂载控制、全 KMI 覆盖。
4. 特色功能：Hide 服务、Umount 服务、内核伪装、路径隐藏、网络隔离。每项展示编号、统一线性图标、标题、简要说明；“内核伪装”保留选中态及 `uname · /proc/version · osrelease` 技术标注。
5. 技术亮点：内核层完成隐藏、LKM/零新增导出符号依赖、ksud 与模块生态兼容、GPL 开源；配 LKM 架构示意图。
6. 页脚：KernelSU 致谢与 GPL 声明、GitHub/使用文档/下载最新版链接及版权。

## 桌面端（>= 1024px）

- 内容最大宽度约 1400px，采用 12 列栅格与大边距。
- Hero 左侧为文字与 CTA，右侧为大型白色模块装置；首屏不堆叠数据卡或徽章。
- “什么是”段采用文本和横向架构示意图的双列构图；四项核心能力为一排等宽、仅以边框区隔的模块。
- 特色功能固定左侧叙事标题，右侧为纵向五行功能清单；激活功能用黑色细框高亮。
- 技术亮点为左侧编号清单、右侧架构图的双列布局，页脚以黑色横线收束。

## 移动端（< 1024px）

- 使用独立的单列流程：顶栏保留标记、菜单按钮与一个主要下载按钮；打开后显示无障碍锚点菜单。
- Hero 改为“标题 → 摘要 → CTA → 装置图”的纵向顺序，避免设备渲染压缩正文；标题根据宽度缩放且不截断。
- 三个 CTA 垂直堆叠或两列换行，所有可点区域最小为 44px。
- 架构示意图在文本之后全宽展示，并允许横向细节在容器中缩放，不裁切重要标注。
- 核心能力改为单列四段；特色功能改为可逐项展开的触控行，默认展示首项并可切换选中态；技术亮点和页脚改为单列。
- 保留纯白、黑色、冷灰和冰蓝的颜色锁定；不因小屏改成深色或暖白。

## 交互与链接

- 导航锚点使用平滑滚动，并在当前区块进入视口时显示活动状态。
- 下载按钮指向 `https://github.com/kusesad-1122/XinovaSU/releases/latest`。
- GitHub 按钮指向 `https://github.com/kusesad-1122/XinovaSU`。
- 使用文档在第一版指向仓库 `website/docs` 内容；实现前将检查是否存在可公开访问的文档站 URL，存在则优先使用该 URL。
- 移动端功能行支持键盘操作、焦点状态及 `aria-expanded`。

## 实现边界

- 独立静态站，不依赖 XinovaSU 主仓库的 Docusaurus 工程。
- 可通过本地静态服务器预览；不引入后端、登录、遥测或下载代理。
- 将抽象图形、图标和装置视觉以 HTML/CSS/SVG 实现，避免把概念图作为网页正文图片。
- 使用用户提供的中文落地页文案作为事实来源；功能对应仓库结构与当前公开信息核对后再写入页面。

## 公开访问要求

- 最终站点必须部署到无需账号、邀请或付费墙即可访问的公开 HTTPS URL；具体托管目标待确认后写入部署配置。
- 页面不根据用户所在地、设备、浏览器或网络身份限制内容；不采集登录信息，也不引入地区性访问门槛。
- 核心文案、样式、字体、图标和交互脚本均随静态站点交付，首屏不依赖第三方 CDN、分析脚本或可能被网络环境拦截的关键资源。
- 保持 JavaScript 失效时的正文和链接可读；图片提供替代文本，导航与移动端功能清单支持键盘操作。
- 以语义化 HTML、足够的文字对比度和 `prefers-reduced-motion` 降低动效作为最低无障碍基线。

## 图层动画

- Hero 右侧的手机/内核装置必须以可独立控制的 HTML/CSS/SVG 图层构建，不使用扁平概念图作为动画主体。图层至少包括：手机机身、屏幕、三张内核模块面板、细技术连线和冰蓝连接节点。
- 桌面端装置默认呈现极轻的上下悬浮与透视倾斜；模块面板以错开的时序向上展开，蓝色节点以低频脉冲表示内核连接。指针悬停时只增加细微的方向性倾斜，不改变阅读区或 CTA 的位置。
- 移动端取消持续的三维倾斜和指针跟随，仅保留一次性的面板入场与低幅度节点脉冲，确保不遮挡正文、不造成布局跳动。
- 功能清单的选中态以边框、冰蓝短线和详情内容的淡入表现；滚动进入视口时可按区块淡入上移一次，不使用无限滚动或夸张的视差。
- 所有动效仅动画 `transform` 与 `opacity`，不驱动布局属性；主要交互保持可键盘触发。启用 `prefers-reduced-motion: reduce` 时，取消悬浮、脉冲、入场位移和任何自动循环，只保留即时状态变化。

## 验收标准

1. 桌面截图在 1440px 宽度下，与已确认概念的纯白、黑色排版、冰蓝状态和疏朗密度一致。
2. 移动截图在 390px 宽度下不发生横向溢出、文字截断、重叠或小于 44px 的主要触控目标。
3. 页面包含所有六个内容区块、五项特色功能和三个可用 CTA。
4. 下载、GitHub、文档链接均使用可验证的目标地址；页面无虚构数据、虚构版本号或未证实的产品承诺。
5. 静态构建或本地预览命令成功，且通过浏览器桌面/移动截图进行视觉检查。
6. 部署后的公开 URL 在无登录的干净浏览器会话中返回成功页面；关键 CSS、字体和图片不依赖被阻断的第三方域名。
7. 桌面端手机装置的各图层可独立动画且不引起布局位移；移动端无持续三维运动；减少动态效果设置下页面不保留自动动画。

## 设计自检

- 未留下 TBD、TODO 或未决定的主要视觉规则。
- 桌面和移动有不同的内容编排与验收条件，响应式范围明确。
- 文案、链接和功能事实均有来源边界；文档 URL 是唯一需要在实现前进行当前状态验证的外部链接。
