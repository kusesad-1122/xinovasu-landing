XinovaSU

内核级 Android Root —— 更深、更隐蔽、更可控

在 Linux 内核层授予与管理 root 权限，而非用户态。基于 KernelSU 深度定制，内建一套面向隐蔽与对抗检测的内核功能。

下载最新版 GitHub 使用文档

什么是 XinovaSU

XinovaSU 是一个基于内核的 Android Root 解决方案，由 KernelSU 二次开发而来。它把 root 的授予、鉴权、模块加载全部下沉到内核层完成——相比传统用户态方案，更难被检测、权限控制更细粒度，并支持 systemless 的模块化系统改造。

支持 Android 12 · 5.10 到 Android 16 · 6.12 共 7 个 GKI KMI。

核心能力

内核级 SU 与鉴权

root 在内核授予，支持按 App 白名单授权、per-app SELinux 域配置。

模块系统 + WebUI

systemless 修改，兼容主流 KernelSU 模块生态，内置 WebUI 入口。

挂载控制

开机自动挂载模块，并可按 App 卸载指定挂载点。

全 KMI 覆盖

跨 7 个 GKI KMI 构建验证，内核模块（LKM）形态加载。

五项功能，留在内核层。

01

Hide 服务

隐藏 Bootloader 解锁 / verified-boot 状态

把一组 ro.boot.* 属性伪装成“已锁定 / green”，骗过读取属性的软件检测。

02

Umount 服务

开机自动卸载指定挂载点

复用内核 try-umount，按 App 生效——被拒的 App 看不到模块挂载。

03

内核伪装

伪装内核版本与构建时间

改写 utsname，使 uname、/proc/version、osrelease 一并报告伪装值。

04

路径隐藏

内核层向指定 App 隐藏文件与目录

命中路径对目标 App 返回 ENOENT，并从目录枚举 getdents 中抹去名字。

05

网络隔离

内核层阻止所选 App 联网

通过 LSM 钩子按 App 拦截外发连接。

更靠近内核，更远离检测。

  • 全部隐藏能力在内核层完成，不依赖易被查杀的用户态特征。
  • LKM 形态，零新增导出符号依赖，跨 7 KMI 编译验证。
  • 保留 ksud 兼容层，兼容 ZygiskNext 与主流模块生态。
  • 开源，GPL 许可。
