OFFICIAL GUIDE · SIMPLIFIED

使用文档

面向 XinovaSU 的中文入门路线。本文按 KernelSU 官方使用文档整理，并保留关键风险提示与官方原文入口；不同设备的分区、KMI 与刷写方式并不相同，请勿跳过准备步骤。

先备份，再刷写。解锁 Bootloader、刷写启动镜像与安装 root 模块都可能造成数据丢失或无法启动。保留与当前固件完全对应的原厂 boot.img 或 init_boot.img，以及可用的恢复路径。

01 / BEFORE YOU START

安装前，请先确认这四件事

Bootloader 已解锁

官方文档明确要求设备能够解锁 Bootloader；无法解锁时，不应继续进行刷写流程。

管理器状态

安装并打开管理器。显示“未安装”表示可继续判断安装方式；显示“不支持”则通常需要自行编译并集成内核。

KMI 与安全补丁

刷写选择以设备内核 KMI 和安全补丁级别为准，不要把 Android 系统版本直接当作内核版本。

原厂镜像备份

保存当前固件对应的原厂启动镜像。它是刷入错误镜像后最直接的恢复材料。

02 / INSTALLATION

选择正确的安装路径

KernelSU 官方将 GKI 与 LKM 作为两种不同工作模式。XinovaSU 的发布内容以实际 Release 说明为准；本页不替代机型专属教程。

  1. 确认设备与 Release 匹配

    先检查设备的 KMI、镜像压缩格式与安全补丁级别，再选用对应版本。镜像不匹配可能导致无法开机。

  2. 优先选择适合的模式

    官方文档建议手机优先考虑 LKM；GKI 会替换内核，LKM 则以可加载模块方式工作。设备或固件的限制可能改变这个选择。

  3. 由管理器修补或安装

    无 root 时使用“选择并修补一个文件”，并只选择当前固件对应的启动镜像；已有 root 时再考虑“直接安装”。Android 13 出厂设备在 LKM 模式下可能操作 init_boot 而不是 boot。

  4. 按机型专属方式刷入

    仅将已修补的镜像刷入管理器或机型文档明确指定的分区。不要把其他设备的命令、分区名或镜像照搬到本机。

  5. 重启并在管理器确认

    首次启动后，确认管理器能够识别内核状态，再向需要的应用逐项授予 root。

03 / MANAGER

把 root 授权收窄到需要的应用

在管理器中按应用授权，避免为未知来源或不需要 root 的应用开启权限。KernelSU 的 App Profile 可针对获得 su 的应用限制 UID、GID、附加组、Capabilities 与 SELinux 规则；这是控制 root 进程权限，而不是替代 Android 应用自身权限管理。

04 / MODULES

模块只从可信来源安装

KernelSU 模块通过管理器安装 ZIP 包。安装后重启并观察启动与基本功能；若模块要求修改 /system，官方资料说明还需要相应的 metamodule 支持。不要在 Recovery 中安装 KernelSU 模块 ZIP，也不要安装来路不明的模块。

05 / OTA & UPDATES

更新前先让系统回到稳定状态

在系统 OTA 或更新 XinovaSU 前，先禁用问题模块并确认当前设备可正常启动。对 A/B 设备，官方资料提供“安装到另一个分区”的流程；是否适用取决于设备与当前安装模式。更新后先验证管理器、root 授权和常用功能，再恢复模块。

06 / RECOVERY

无法启动时的恢复顺序

  1. 强制重启一次

    若刚更新或安装模块后无法启动，官方救砖文档说明模块 AB 更新机制可能在重启后回滚并禁用本次问题模块。

  2. 进入安全模式

    可使用系统自带安全模式；KernelSU 内置安全模式的触发方式是开机第一屏后连续按音量下键超过三次。进入后卸载问题模块。

  3. 刷回原厂启动镜像

    因错误 boot 镜像、压缩格式不匹配或内核不适配而无法启动时，按设备文档刷回此前备份的原厂镜像。

  4. 必要时恢复完整官方系统

    若无法通过以上路径恢复，使用设备官方的完整固件恢复流程或联系售后。不要继续反复刷写未知镜像。
